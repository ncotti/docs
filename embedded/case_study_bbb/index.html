    Case study: BBB device tree

    Let's analize the BBB device tree, and see what we find. The idea is to compare the hardware description with the PCB schematic, the datasheet and the associated drivers.

    Disclaimer: for simplicity and learning purposes, several parts of the device tree have been simplified. For example, phandles have been expanded or certian parts have been ommited.

    There are several include files, so this is just a collection of them.

    General overview

    How do you read a device tree? Where do you start to describe your hardware.

    These are questions rooted on computer architecture theory, which I am not an expert in. However, my two cents are the following:

    Start by identifying the CPU and the CPU interconnect bus (AMBA, AXI, OCP, APB, etc). In any CPU architecture, everything is connected to a trunk interconnect, with a memory address.

    By looking at the functional block diagram of the AM335x processor, you can see that the interface between the ARM Cortex-A8 and all other external peripherals are the L3 and L4 interconnects.

    When digging a little deeper, you can read that there is an AXI to OCP translator, and that the L3 and L4 are also OCP buses.

    CPU node

    The CPU node declares a single CPU. The compatible method does not point to a driver, but rather one of the possible CPU architectures specified here: "Documentation/devicetree/bindings/arm/cpus.yaml".
